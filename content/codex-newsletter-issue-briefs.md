# Codex Newsletter Issue Briefs

## How To Use This File

Use this file as the drafting brief for the 12-issue series. Each issue brief is intentionally detailed enough to guide a full newsletter draft and prevent shallow explanation.

For every issue:

1. Start with the story beat.
2. Teach the concept from first principles.
3. Keep explaining until a non-technical reader could restate it simply.
4. Use the workflow steps exactly as written.
5. Add screenshots and one concrete artefact.
6. Support factual claims with official sources.

## Issue 1

- Issue title: What Codex Actually Is
- Main concept: Codex as an execution-oriented agent
- Secondary concepts:
  - Ask vs Code
- Reader outcome: The reader can explain the difference between discussing work and assigning work
- Story beat: A new inbound lead appears. The operator needs a qualified follow-up prepared, not a vague brainstorming session.
- Plain-English definition: Codex is a system you can ask to carry out real tasks, not only answer questions.
- What problem this solves: Too much time is lost when people use AI only to talk about work instead of moving work forward.
- What breaks when this is missing: The operator gets suggestions but no reliable execution path.
- Mechanism explanation:
  1. Explain the difference between conversation and execution.
  2. Explain what a task is.
  3. Explain why tasks need boundaries, outputs, and review.
  4. Restate the whole concept in simple everyday language.
- Workflow steps:
  1. Identify the lead.
  2. Define the task objective.
  3. State the desired output.
  4. State the constraints.
  5. Decide what will be reviewed before action.
- Screenshot plan:
  - Ask-style prompt
  - Code-style task
- Artefact plan:
  - first task definition
- Action checklist:
  - pick one recurring task
  - rewrite it as an executable task
  - identify the review point
- CTA: Reply with the most repetitive task in your workflow.
- Evidence requirements:
  - Codex positioning
  - Ask vs Code distinction
